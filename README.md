# Выполнение ДЗ к курсу OTUS "Golang Developer: Basic"

## Знакомство и начало работы с Go //ДЗ №1

Описание/Пошаговая инструкция выполнения домашнего задания:
Установить Go SDK;
Установить и настройка IDE;
Создать учетную запись на Github;
Установить Git, Github Desktop;
Создать пустой публичный проект для домашних заданий;
Cклонировать проект на свое устройство;
Попробовать написать первые строчки в IDE кода и запустить.



## Работа с Git //ДЗ №2

Описание/Пошаговая инструкция выполнения домашнего задания:
Создание базовой ветки проекта, например: main
Создание новой ветки hw-02 от main
Добавление кода, выполнение комита в новой ветке
Запушить в удаленный репозиторий новую ветку
Создать pull request и отправить его на проверку
Получить комментарий от ментора и исправить его
Получить подтверждение выполненной работы от ментора и смерджить результат
Обновить локальную базовую ветку (fetch, pull)
Проверить, что все изменения комита в основной ветке.


## Управляющие конструкции //ДЗ №3


Описание/Пошаговая инструкция выполнения домашнего задания:
Напишите программу, создающую строку, содержащую решётку 8х8, в которой линии разделяются символами новой строки;
На каждой позиции либо пробел, либо #;
В результате должна получиться шахматная доска;
Сделайте размер доски переменным, чтобы можно было создавать доски любого размера;


## Пакеты и функции// ДЗ №4

Описание/Пошаговая инструкция выполнения домашнего задания:
Продолжаем ДЗ-3 «Шахматная доска».

Перед началом запрашиваем размерность поля.
Запрашиваем ввод имен игроков.
Добавляем номера строк и буквы для столбцов.
Расставляем на доске ч/б символы шахматных фигур из юникода.
«Лишние» фигуры не добавляем, недостающие — повторяем.
Указываем сбоку от игрового поля имена игроков.
Опционально — использовать ANSI-цвета, например, для фона доски.


## Пользовательские типы, структуры и методы //ДЗ №5

Продолжаем ДЗ-4 «Расставляем фигуры».
Добавляем следующие сущности:
— игроки
— игры
— ходы
— игровое поле

Предложить самим состав указанных выше сущностей.

Создать отдельные пакеты в internal/model, которые будут содержать все структуры, необходимые для реализации бизнес-логики вашего приложения.
С помощью структур и их полей необходимо описать все сущности, которые будут использоваться в вашем проекте, и сгруппировать их по пакетам, чтобы сильно связанные структуры в одном пакете и слабо связанные в другом.
Попробовать реализовать в структуре приватные переменные, которые можно отобразить снаружи пакета только с помощью метода структуры и которые можно изменить тоже только через метод.
